# Codenames

This repository contains instructions I created to help a college student refactor one of their [class projects](https://github.com/LKieser/codenames) (after the class finished) and port it from Python to C. A lot of the material was inspired by [write ups](https://github.com/EricSchrock/software-engineering-intro) I had previously created for structuring a software engineering mentorship.


### Steps

1. [Set up environment, version control, and baseline](https://github.com/EricSchrock/codenames/blob/master/step01.md)
2. [Refine game loop](https://github.com/EricSchrock/codenames/blob/master/step02.md)
3. [Add tests](https://github.com/EricSchrock/codenames/blob/master/step03.md)
4. [Port "state update" type functions to C](https://github.com/EricSchrock/codenames/blob/master/step04.md)
5. [Port unit tests to C](https://github.com/EricSchrock/codenames/blob/master/step05.md)
6. [Port remaining code to C](https://github.com/EricSchrock/codenames/blob/master/step06.md)
7. [Add auto-formatting, static analysis, and dynamic analysis](https://github.com/EricSchrock/codenames/blob/master/step07.md)
8. [Add documentation with Markdown and flow charts](https://github.com/EricSchrock/codenames/blob/master/step08.md)
9. [Demo TUI for user feedback](https://github.com/EricSchrock/codenames/blob/master/step09.md)
10. [Port from TUI to GUI](https://github.com/EricSchrock/codenames/blob/master/step10.md)
11. [Demo GUI for user feedback](https://github.com/EricSchrock/codenames/blob/master/step11.md)
12. [Add project to professional portfolio](https://github.com/EricSchrock/codenames/blob/master/step12.md)
